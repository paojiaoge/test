# 管100台服务器的运维，终于不用半夜爬起来了

我有个朋友做运维，管着公司六十多台服务器。

上个月他跟我吐槽，说凌晨三点被电话叫醒，某台机器磁盘满了，服务挂了。他迷迷糊糊打开电脑，SSH连上去，df -h看了一眼，发现是日志文件没清理。删了几个大文件，重启服务，前后折腾四十分钟。

这种事他一个月至少碰两三次。

问题是，这活儿一点技术含量都没有。就是登上去敲几条命令的事。但因为没有一个统一的管理工具，每次都得手动SSH，手动排查，手动处理。

他说，最崩溃的不是半夜起来干活，而是干的活太简单了，简单到你觉得不值得为它醒一次。

## 运维的痛，不是技术难，是重复多

做过运维的人都知道，日常工作里真正需要动脑子的事不多。大部分时间花在这些地方：

批量更新一个配置，要一台一台SSH上去改。部署一个脚本，要一台一台传文件、改权限、执行。查个日志，要记住哪台机器跑的哪个服务。

有人说，写个脚本批量跑不就行了？

行，但你得维护这些脚本。机器加了要改，IP变了要改，密码换了要改。时间一长，脚本比服务器还难管。

Ansible、SaltStack这些工具确实能解决一部分问题，但学习成本不低，配置也挺麻烦。对于中小团队来说，经常是工具还没配好，活儿已经手动干完了。

## EasyShell：一个开源的服务器管理平台

最近发现一个开源项目叫EasyShell，试了一下，觉得思路挺对的。

它做的事情说白了就一句话：给你一个Web界面，让你在浏览器里管所有服务器。

具体能干什么：

**批量执行脚本**。选一批机器，写一段Shell，点执行，所有机器同时跑，实时看每台机器的输出日志。不用一台一台SSH了。

**Web终端**。直接在浏览器里打开一个终端，SSH连到任意一台机器。不用装客户端，不用记IP和密码，点一下就连上了。

**AI运维助手**。这是我觉得最有意思的部分——你可以用自然语言告诉它你要干什么，它帮你生成命令并执行。比如"帮我查一下所有机器的磁盘使用率"，它自己去跑命令，把结果汇总给你。

**脚本库**。社区共享的运维脚本，别人写好的直接拿来用，不用自己从头写。

**安全管控**。危险命令会被拦截，需要审批才能执行。这对团队管理来说挺重要的，不怕新人误操作把生产环境搞崩。

## 5分钟部署，不是吹的

我自己试了一下部署，确实快。

前提条件就一个：装了Docker。

三步搞定：

```bash
# 第一步：克隆代码
git clone https://github.com/easyshell-ai/easyshell.git
cd easyshell

# 第二步：复制配置文件
cp .env.example .env

# 第三步：一键启动
docker compose up -d
```

跑完之后，打开浏览器访问 `http://localhost:18880`，用默认账号 `easyshell / easyshell@changeme` 登录就行了。

它会启动四个容器：管理服务端、Web前端、MySQL、Redis。全部容器化，不用你装Java、不用装Node.js，Docker一把梭。

然后在Web界面里添加你要管理的服务器，它会自动SSH过去部署一个轻量级的Agent（用Go写的，很小），之后这台机器就纳入管理了。

## AI运维这块，值得单独说说

EasyShell接入了AI能力，支持OpenAI、DeepSeek或者任何兼容OpenAI接口的模型。

配置方法：进系统管理 → AI设置，填上你的API Key和模型地址就行。

配好之后，你可以在界面里用自然语言下指令。比如：

- "检查所有服务器的内存使用情况"
- "找出磁盘使用率超过80%的机器"
- "在所有Web服务器上重启Nginx"

它会把你的自然语言转成具体的Shell命令，你确认之后再执行。不是那种"AI自动帮你干完你都不知道干了啥"的模式，而是AI建议、人类确认、再执行。这个设计我觉得比较靠谱，运维这种事，还是得人把关。

回到开头那个场景——凌晨三点磁盘满了。如果用EasyShell，你甚至可以提前设置定时任务，让它每天自动检查磁盘使用率，超过阈值自动清理日志。根本不需要人半夜爬起来。

## 技术栈一览

给技术同学看一下它的架构：

| 组件 | 技术栈 | 说明 |
|------|--------|------|
| Server | Java 17 + Spring Boot 3.5 | 中心管理服务 |
| Agent | Go 1.23 | 部署在每台被管理机器上，很轻量 |
| Web | React 19 + Vite 7 + Ant Design 6 | 前端界面 |

Server和Agent之间用HTTP做注册和心跳，用WebSocket做实时日志流。架构不复杂，代码开源，想改可以自己改。

## 适合谁用

说实话，如果你只管两三台服务器，SSH挨个连也不费事，没必要上这个。

但如果你管的机器超过十台，或者团队里有多个人需要操作服务器，EasyShell能省不少事：

- 不用记一堆IP和密码
- 不用写一堆批量脚本然后维护它们
- 新人来了给个账号就能用，不用教半天
- 操作有审计日志，出了问题能追溯

项目地址：https://github.com/easyshell-ai/easyshell
文档地址：https://docs.easyshell.ai

开源免费，自己部署，数据在自己手里。对中小团队来说，这可能比花钱买商业运维平台实在。
